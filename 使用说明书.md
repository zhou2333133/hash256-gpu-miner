# HASH256 GPU 挖矿器完整使用说明书

更新时间：2026-05-11

## 1. 安全提醒

这个程序会在开启自动发送后真实支付 Ethereum Mainnet Gas。请只使用新钱包，只放少量 ETH 作为 Gas，不要使用主钱包，不要导入助记词。

程序硬编码安全边界：

- 只允许目标合约：`0xAC7b5d06fa1e77D08aea40d46cB7C5923A87A0cc`
- 只允许函数：`mine(uint256)`
- 函数选择器：`0x4d474898`
- 交易 `value` 永远强制为 `0`
- 默认只模拟：`DRY_RUN=true`
- 默认不自动发交易：`AUTO_SEND=false`
- 私钥只从 `.env` 的 `MINER_PRIVATE_KEY` 读取

## 2. 进入 WSL 项目目录

在 WSL Ubuntu 里执行：

```bash
cd "/mnt/c/Users/Administrator/Documents/New project 4/hash256-gpu-miner"
```

## 3. 安装 Node.js 依赖

如果 WSL 已有 Node.js：

```bash
node -v
npm -v
npm install
```

如果没有 Node.js：

```bash
sudo apt update
sudo apt install -y nodejs npm build-essential
node -v
npm -v
npm install
```

推荐 Node.js 版本 `20+`。

## 4. 检查 NVIDIA / CUDA

先确认 WSL 能看到显卡：

```bash
nvidia-smi
```

检查 CUDA 编译器：

```bash
nvcc --version
```

如果没有 `nvcc`，先安装 CUDA Toolkit。Ubuntu 常见方式：

```bash
sudo apt update
sudo apt install -y nvidia-cuda-toolkit build-essential
nvcc --version
```

如果 `nvidia-smi` 不可用，需要先在 Windows 安装支持 WSL 的 NVIDIA 驱动。

## 5. 编译 CUDA worker

```bash
npm run build:cuda
```

确认产物存在：

```bash
ls -lh gpu/cuda_miner
```

如果 CUDA 编译失败，程序仍可用 CPU 回退模式运行，但 CPU 只适合验证流程，不适合高效挖矿。

## 6. 配置 .env

复制模板：

```bash
cp .env.example .env
nano .env
```

最小配置只需要一行：

```env
MINER_PRIVATE_KEY=0x你的新钱包私钥
```

保存 nano：

- `Ctrl+O`
- 回车
- `Ctrl+X`

`.env` 已被 `.gitignore` 忽略，不会进入源码包。

## 7. 先用只模拟模式运行

默认就是只模拟：

```bash
npm run dev
```

只模拟模式会：

- 读取钱包地址和 ETH 余额
- 读取链上 epoch、difficulty、challenge
- 启动 CUDA 搜索 nonce
- 找到候选 nonce 后执行 `eth_call` 模拟
- 不发送真实交易

终端会显示中文实时面板。

## 8. 开启自动发送

确认你已经使用新钱包，并且钱包只放少量 ETH 作为 Gas。

编辑 `.env`：

```bash
nano .env
```

最小自动发送配置：

```env
MINER_PRIVATE_KEY=0x你的新钱包私钥
DRY_RUN=false
AUTO_SEND=true
```

然后运行：

```bash
npm run dev
```

开启后流程是：

1. GPU 搜索 nonce
2. 找到候选 nonce
3. 本地复核 hash
4. 检查 epoch 是否仍有效
5. 检查该 nonce 是否已提交过
6. `eth_call` 模拟 `mine(uint256)`
7. 估算 Gas
8. 检查 Gas 上限
9. 检查交易安全边界：目标合约、selector、value、chainId、type
10. 发送 EIP-1559 type 2 交易

## 9. 可选配置

默认已经内置多个公共主网 RPC：

```text
https://ethereum.publicnode.com
https://ethereum-rpc.publicnode.com
https://eth-mainnet.public.blastapi.io
```

程序会在链上读取、模拟、估 Gas、广播交易时自动切换 RPC。如果你有自己的 RPC，可以加；你的 RPC 会排在默认公共 RPC 前面：

```env
RPC_URL_PRIMARY=https://你的主RPC
RPC_URL_BACKUP_1=https://你的备用RPC1
RPC_URL_BACKUP_2=https://你的备用RPC2
```

交易发送时，程序会先本地签好 raw transaction，再把同一笔交易轮流广播到多个 RPC。这样不会依赖网页钱包弹窗，也不会因为单个 RPC 掉线就卡在“无法签名”。

Gas 安全上限默认值：

```env
MAX_FEE_GWEI_CAP=3
MAX_PRIORITY_FEE_GWEI_CAP=0.5
GAS_MULTIPLIER=1.15
MAX_GAS_LIMIT=150000
MAX_ETH_PER_TX=0.0002
MAX_ETH_PER_HOUR=0.002
```

如果主网 Gas 高于上限，程序会继续挖矿，但不会自动发送交易。

程序会跟随市场 Gas 自动计算 EIP-1559：

```text
maxPriorityFeePerGas = 当前建议 priority * GAS_MULTIPLIER
maxFeePerGas = baseFee * 2 + priorityFee
```

但它永远不会超过你设的上限。如果链上 base fee 已经是 `7 gwei`，而你仍然设置：

```env
MAX_FEE_GWEI_CAP=3
```

程序会阻止发送，这是安全设计。想在较高 Gas 市场下更容易广播，可以提高上限，例如：

```env
MAX_FEE_GWEI_CAP=50
MAX_PRIORITY_FEE_GWEI_CAP=3
GAS_MULTIPLIER=1.30
MAX_ETH_PER_TX=0.006
MAX_ETH_PER_HOUR=0.012
```

如果 `gasLimit=150000`、`maxFee=50 gwei`，理论最大成本是 `0.0075 ETH`，所以上面的 `MAX_ETH_PER_TX=0.006` 仍会在单笔成本过高时拦截。实际花费通常低于最大值，但钱包必须能承受你设定的上限。

`MAX_FEE_GWEI_CAP=50` 不是固定用 50 gwei 发，而是允许程序跟随链上 `baseFee * 2 + priorityFee`，最高不超过 50 gwei。真正是否发送，还会受 `MAX_ETH_PER_TX` 和 `MAX_ETH_PER_HOUR` 限制。

CUDA / 批次配置：

```env
CUDA_DEVICE_ID=0
NONCE_BATCH_SIZE=1000000000
```

`NONCE_BATCH_SIZE` 是每次交给 CUDA worker 的 nonce 数量。批次越大，重启 worker 和读取链上状态的开销越小；批次太大，epoch 变化时会多浪费旧任务时间。以 `130 MH/s` 左右估算：

- `100000000` 约 0.7 秒一批
- `1000000000` 约 7.5 秒一批
- `10000000000` 约 75 秒一批，不推荐，epoch 快结束时浪费较大

建议默认用 `1000000000`。如果你手动设置了 `.env`，`.env` 会覆盖程序默认值。

### 常见合约错误

`0x4992976a` = `BlockCapReached()`，意思是当前区块的 mint 名额已满。新版程序会先尝试 `pending` 模拟；如果 pending 仍不可行，会保留候选 nonce，在同一 epoch 的下一个区块继续重试。若 epoch 已经切换，旧 nonce 的 challenge 已经改变，无法再补救。

终端面板：

```env
TERMINAL_DASHBOARD=true
```

关闭中文实时面板，恢复普通滚动日志：

```env
TERMINAL_DASHBOARD=false
```

## 10. 终端面板字段说明

- `自动发送已开启`：找到有效 nonce 后会尝试自动提交。
- `只模拟，不发送`：只挖和模拟，不发真实交易。
- `函数选择器`：必须是 `0x4d474898`，对应 `mine(uint256)`。
- `转账金额`：必须是 `0`。
- `目标合约`：必须是 HASH256 合约。
- `钱包`：当前挖矿地址，也是提交交易地址。
- `ETH 余额`：该钱包可用 Gas 余额。
- `区块`：当前 Ethereum 主网区块。
- `Epoch`：合约按 `block.number / 100` 计算的挖矿轮次。
- `本轮剩余区块`：当前 epoch 还剩多少区块。
- `难度/目标`：hash 数值必须小于这个 target。
- `挑战值`：绑定当前钱包、合约、chainId、epoch。
- `计算后端`：`cuda` 表示 GPU，`cpu` 表示 CPU 回退。
- `算力`：当前 H/s 和 MH/s。
- `GPU 利用率`：当前 worker 未读取时显示 `未读取`，可以用 `nvidia-smi` 查看。
- `累计扫描`：本次运行累计尝试 nonce 数量。
- `候选 Nonce`：找到的候选 nonce。
- `提交结果`：模拟、拦截或发送结果。
- `交易哈希`：真实发送后显示 tx hash。

## 13. 常见问题

### AUTO_SEND=true 但启动失败

如果看到：

```text
AUTO_SEND=true requires DRY_RUN=false
```

说明 `.env` 缺少：

```env
DRY_RUN=false
```

自动发送必须显式同时配置：

```env
DRY_RUN=false
AUTO_SEND=true
```

### GPU 显示未读取

面板里的 `GPU 利用率：未读取` 不是错误。当前 CUDA worker 不读取利用率，只显示挖矿算力。用下面命令查看真实 GPU 状态：

```bash
watch -n 1 nvidia-smi
```

### 找到 nonce 但没有发交易

常见原因：

- 仍在 `DRY_RUN=true`
- `AUTO_SEND=false`
- epoch 已变化，nonce 失效
- `eth_call` 模拟失败
- Gas 超过 `MAX_FEE_GWEI_CAP`
- 单笔成本超过 `MAX_ETH_PER_TX`
- 最近 1 小时成本超过 `MAX_ETH_PER_HOUR`

如果你在网页钱包里曾经挖到 nonce，但因为 MetaMask/OKX/RPC 没连上导致没有成功签名或广播，可以尝试手动救援提交。前提是：

- 必须是同一个钱包地址挖到的 nonce
- 当前 epoch 还没有变化
- 该 proof 没有链上使用过

命令：

```bash
npm run submit-nonce -- 你的十进制nonce
```

如果要真实发送，`.env` 必须有：

```env
DRY_RUN=false
AUTO_SEND=true
```

工具会先本地复核 hash，再查 `usedProofs`，再 `eth_call` 模拟，通过后才会按 Gas 上限发送 `mine(uint256)`。

### 交易失败

失败交易也会扣 Gas。常见原因：

- epoch 在上链前变化
- 其他人同区块竞争成功
- 当前区块达到合约 `MAX_MINTS_PER_BLOCK`
- Gas 太低导致 pending 太久

程序不会无限重试同一个 nonce。
